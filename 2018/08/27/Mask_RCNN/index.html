Mask_RCNN源码学习

安装和部署

详细步骤参考源码主页

  1. Install dependencies

  2. Clone this repository

  3. Run setup from the repository root directory

  4. Download pre-trained COCO weights (mask_rcnn_coco.h5) from the releases page.

  5. (Optional) To train or test on MS COCO install pycocotools from one of these repos. They are forks of the original pycocotools with fixes for Python3 and Windows (the official repo doesn’t seem to be active anymore).

    • Linux: https://github.com/waleedka/coco
    • Windows: https://github.com/philferriere/cocoapi. You must have the Visual C++ 2015 build tools on your path (see the repo for additional details)

Step 3. Training

Train in two stages:

  1. Only the heads. Here we’re freezing all the backbone layers and training only the randomly initialized layers (i.e. the ones that we didn’t use pre-trained weights from MS COCO). To train only the head layers, pass layers='heads' to the train() function.

  2. Fine-tune all layers. For this simple example it’s not necessary, but we’re including it to show the process. Simply pass layers="all to train all layers.

通过train函数可以看出，调用该函数时可以任意指定需要训练的层。

Issues 收集

1.关于pycharm 和 jupyter notebook 导入模块差异的问题

在学习Mask_RCNN源码的时候，遇到一个有关导入模块差异的问题，硬是浪费了不少时间。
  • 使用pycharm(不建议使用)

    如果通过pycharm运行jupyter notebook, pycharm会自动将项目的根目录加入到sys.path中，这是因为pycharm中有加入根目录的选项，如下图所示。

使用pycharm操作还有个问题：有些模块的类无法识别（在pycharm中），但在浏览器中没有问题。

比如在pycharm中使用 os.getcwd()无法识别当前目录，而是识别的根目录。但如果通过pycharm启动jupyter然后通过浏览器访问便没有问题，但这样做pycharm和jupyter的同步是个很大的问题，经常在浏览器运行后无法同步到文件，导致浏览器运行的结果丢失。

  • 直接在终端运行jupyter notebook（推荐使用）

    直接使用jupyter notebook并使用浏览器有以下几个问题：

    1. jupyter notebook不会自动在sys.path中加入根目录，导致很多模块导入出现问题。
    2. jupyter 默认不开启代码补全功能，需要修改配置文件或者下载插件。
    对于第一个问题：我们可以通过sys.path.append(ROOT_DIR)手动添加需要的目录。但是这些目录是添加到sys.path列表的最后，导入模块的优先级是按照列表排序的，所以如果需要导入的模块和排序靠前的某个lib模块重合便会导致无法识别模块。比如Mask_RCNN根目录下有一个utils.py，而排在列表前的'D:\\programSoft\\Anaconda3\\lib\\site-packages“也包含utils.py, 所以无法找到utils.Dataset类。

    解决办法：通过sys.path.insert()将根目录插入列表最前面。

    对于第二个问题：安装一个插件后，勾选Hinterland即启用了代码自动补全。该插件还有很多的功能，可以进一步了解一下。

2.关于源码keras版本兼容性问题

[# issue 662](https://github.com/matterport/Mask_RCNN/pull/662): 源码keras版本是2.0.8, keras.engine.topology已经被弃用，将model里的topology换成saving便可解决问题。
